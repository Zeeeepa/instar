# Upgrade Guide — v0.12.14

<!-- bump: patch -->

## What Changed

- **Coherence monitor notifications now show check names on first line.** Previously, the batched notification only showed "Coherence: 1 issue(s):" with the actual failed check names truncated (they were on the second line, which formatDigest drops). Now the first line reads e.g. "Coherence: config-parity failed (1 issue)" so the attention topic message is immediately informative.

- **External process alerts reduced to once per 24 hours.** The OrphanProcessReaper was generating repeated alerts about external Claude processes (VS Code, terminal sessions) every digest cycle. These are normal processes and the alerts were noise. Now alerts fire at most once per 24h with a summary count and total memory, not individual PID listings.

## What to Tell Your User

- **Clearer attention messages**: "Coherence alerts in the attention topic now tell you *what* failed, not just that something failed. No more cryptic 'Coherence: 1 issue(s):' with no details."
- **Less noise from external process alerts**: "You won't see repeated warnings about VS Code/terminal Claude processes anymore — those are normal and now only surface once a day."

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|
| Informative coherence alerts | Automatic — attention topic messages now include check names |
| Quieter external process reporting | Automatic — 24h cooldown on external process alerts |
