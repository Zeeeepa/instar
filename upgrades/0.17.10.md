# Upgrade Guide — v0.17.10

<!-- bump: patch -->

## What Changed

### /new Command Fix — Session Spawning

The `/new` command in Telegram was broken: it created a new topic but didn't spawn a session in it, resulting in a dead topic with no active agent.

**Root cause**: Two `/new` handlers existed — one in `TelegramAdapter.handleCommand()` (which only created a topic) and one in the server's `onTopicMessage` callback (which created a topic AND spawned a session). The adapter handler intercepted `/new` first and returned `true`, preventing the server handler from ever running.

**Fix**: Removed the incomplete `/new` handler from `TelegramAdapter.handleCommand()`. The server-level handler now processes `/new` correctly — creates the topic, spawns an interactive session, links them together, and confirms in both the source and new topic.

## What to Tell Your User

- The /new command works now. Typing /new followed by a topic name creates a new topic with a live session. Previously it created empty topics with no session.
