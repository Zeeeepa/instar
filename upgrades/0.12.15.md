# Upgrade Guide — v0.12.15

<!-- bump: patch -->

## What Changed

Notification messages sent to Agent Attention topics have been completely rewritten for clarity:

1. **NotificationBatcher.formatDigest**: Removed the terse `--- Summary (N items) ---` headers and `CATEGORY:` grouping. Messages now show their full content in chronological order, making each notification self-contained and readable.

2. **Coherence Monitor notifications**: Instead of `Coherence: output-sanity, readiness-auth-token failed (2 issues)`, each failure is now translated into a narrative explanation with a suggested action (e.g., "Security: API Unprotected — Your agent's API has no authentication token... Reply 'fix auth' to generate one.").

3. **Orphan Process Reaper notifications**: Instead of `1 external Claude process(es) detected (231MB total). Use API /monitoring/processes to inspect.`, now says something like "Found 1 Claude process running outside your agent (using 231MB of memory). This is usually from VS Code... Reply 'clean processes' to free up the memory."

4. **System Reviewer (Doctor) notifications**: Instead of `[DOCTOR] CRITICAL: Lifeline Supervisor failed`, now provides context like "Your agent's automatic restart system (lifeline) has stopped working... Reply 'fix lifeline' to restart it."

## What to Tell Your User

- **Clearer notifications**: "Your Agent Attention messages are now easier to understand. Instead of technical codes, you'll see plain-language explanations of what happened and what you can do about it."

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|
| Narrative notifications | Automatic — all system notifications now use plain language |
| Suggested actions in notifications | Reply to Agent Attention with the suggested command |
